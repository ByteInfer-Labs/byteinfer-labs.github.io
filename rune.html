How it works

A rule names a pattern of operators, then the commands that change the matched site. Three moves cover most work.

One rule, thirteen ops gone

Frameworks often spell a single operation as a tree of primitives the runtime never fuses. A rule inserts the real operator; the primitive subtree loses its consumers and is pruned.

pruned inserted
Before — TensorFlow LayerNorm
After — one fused operator

The pattern matches the thirteen-operator idiom and inserts a single LayerNormalization. Once its output feeds the downstream consumer, the primitive subtree is dead and pruned automatically.

On BERT-Squad this fusion, applied at every LayerNorm site, runs measurably faster than ONNX Runtime at its most aggressive optimization level — and the outputs stay bit-identical to the original.

Every rewrite is checked

Before a single node changes, Rune simulates the plan and asks one question per site: if this runs, is the result still a complete graph?

The validity gate

A handful of invariants define complete. A rewrite that would break one is refused at that site — with the invariant, the node, and the port named — rather than producing a model that crashes three transforms later. This is what makes rule authoring safe to hand to an agent: a bad rewrite is a machine-readable refusal, the way a compiler error is to code.

I1Every required input has a producer behind it.
I2No surviving node reads a deleted tensor.
I3Every graph output is still produced.
I4The graph stays acyclic.
I5Node arity matches the operator schema.
I7Attributes are known to the target runtime.

A rewrite is a plan, checked before it runs

The gate answers one question per site: if this plan runs, is the result still a complete graph? Five invariants define complete, and a rewrite that would break one is refused where it stands — the invariant, node, and port named.

Because the gate simulates the plan on a shadow copy of the graph before any node is written, a bad rewrite costs nothing — it returns as a refusal naming the invariant it would have broken, not as a model that loads and then produces wrong numbers three transforms downstream.
